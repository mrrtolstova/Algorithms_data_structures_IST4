Задача: 

Реализовать игру Хидоку.

Головоломка представляет собой разделенную на блоки прямоугольную сетку, некоторые клетки первоначально содержат числа.
Головоломка придумана израильским математиком Гиором Бенедеком (Gyora Benedek). Название в переводе с иврита означает “моя головоломка”.
Задача – заполнить сетку числами. Числа “вписываются” последовательно (1, 2, 3, 4 и т. д.) и только в соседние клетки, соединенные между собой по горизонтали, 
по вертикали или по диагонали. Первое и последнее число обязательно присутствуют в сетке.

Необходимо:

- реализовать выбор размера игрового поля пользователем
- реализовать вывод решенной игры на экран
- реализовать передачу введенных данных в обработку другим классом
- реализовать обработку исключений

Процесс:

(27.03.2020) + создан первый прототип игрового поля как образец
(27.03.2020) + решено, что поля будут иметь строковый формат и забираться потоковым вводом
(27.03.2020) + реализовано преобразование игрового поля в квадрат (или другая фигура) из строки (через разделители * и .)
(28.03.2020) + реализован алгоритм получения "соседей" координаты
(28.03.2020) + реализован поиск "1" для начала игры
(28.03.2020) + реализован поиск последующего хода
(28.03.2020) + реализован вывод на экран решенной игры
(29.03.2020) + разработаны и учтены все исключения (ввода, неправильного формата строки)

Реализованные функции:

• main - хранит в себе ширины игровых полей, отвечает за ввод данных и последующую передачу их на обработку
• hSolver - показывает координаты "соседей" текущий координаты
• solve - функция поиска "решения", хранит массив структур arr и булевый массив weHave. Ищет следующий ход
• solveIt - функция для поиска возможности хода
• findStart - функция поиска первого значения, от которого будет начинаться игра 
• getNeighbours - функция взятия двух соседних квадратов игрового поля
• search - функция поиска (вычисление предполагаемого и его реализация)
